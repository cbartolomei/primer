🌐 HTTP Servers

Building production HTTP services with Go's standard library — routing, middleware, templating, SQL databases, sessions, and TLS. No framework required.

🔌

net/http — The Server Model

Go's net/http package is a production-grade HTTP server with no external dependencies. Understanding its two core abstractions — Handler and ServeMux — is all you need to build real services.

The Handler Interface

Everything in net/http implements one interface:

type Handler interface {
    ServeHTTP(ResponseWriter, *Request)
}

// ResponseWriter lets you write status + headers + body
// *Request carries the incoming data: method, URL, headers, body

http.HandlerFunc is a function adapter — it lets any function with the right signature be used as a handler without creating a named type:

// HandlerFunc is just a type alias with ServeHTTP defined on it
func greet(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "hello")
}

mux.Handle("/greet", http.HandlerFunc(greet)) // explicit conversion
mux.HandleFunc("/greet", greet)               // HandleFunc does the conversion for you

ServeMux and Go 1.22 Routing

Before Go 1.22, ServeMux only matched path prefixes. Go 1.22 added method-based routing and wildcard path segments — closing the gap with third-party routers for most use cases:

mux := http.NewServeMux()

// Go 1.22+ patterns: "METHOD /path/{wildcard}"
mux.HandleFunc("GET /snippets/{id}", snippetView)
mux.HandleFunc("POST /snippets", snippetCreate)
mux.HandleFunc("DELETE /snippets/{id}", snippetDelete)

// Trailing slash = subtree match (any path under /static/)
mux.Handle("GET /static/", http.FileServer(http.Dir("./ui/static")))

// Root pattern matches everything not matched above
mux.HandleFunc("/", notFound)

// Extract wildcard value from the path
func snippetView(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id") // "id" matches {id} in the pattern
    // ...
}
Patterns without a method prefix match any method — useful for a catch-all. Patterns with a method prefix take priority over method-less patterns at the same path.

Configuring the Server

Never use http.ListenAndServe in production — it uses the default mux and has no timeouts. Always create an explicit http.Server:

srv := &http.Server{
    Addr:    ":4000",
    Handler: mux,

    // Timeouts protect against slow clients and resource exhaustion.
    // Zero means no timeout — a single slow client can block a goroutine forever.
    ReadTimeout:  5 * time.Second,   // time to read the full request, including body
    WriteTimeout: 10 * time.Second,  // time to write the full response
    IdleTimeout:  60 * time.Second,  // keep-alive connections sit idle for at most this long

    // Structured logger (Go 1.21+)
    ErrorLog: slog.NewLogLogger(logger.Handler(), slog.LevelError),
}

log.Fatal(srv.ListenAndServe())

Project Structure

The layout that Let's Go and most production Go web services converge on:

myapp/
├── cmd/
│   └── web/
│       ├── main.go        // wires everything together: DB, handlers, server
│       ├── handlers.go    // HTTP handler functions
│       ├── middleware.go  // middleware functions
│       └── routes.go      // mux setup
├── internal/
│   └── models/
│       ├── snippets.go    // database model + queries
│       └── users.go
├── ui/
│   ├── static/            // CSS, JS, images
│   └── html/
│       └── base.html      // templates
├── go.mod
└── go.sum

The internal/ directory is enforced by the Go toolchain — packages inside it can only be imported by code in the parent tree. Use it for database models and business logic you don't want to expose as a public API.

Dependency Injection via Application Struct

Handlers need shared dependencies: a logger, a DB pool, a session manager. The idiomatic Go approach is an application struct — not global variables, not a framework container:

// main.go
type application struct {
    logger   *slog.Logger
    snippets *models.SnippetModel
    sessions *scs.SessionManager
}

// Handlers become methods on *application, giving them access to all deps
func (app *application) snippetView(w http.ResponseWriter, r *http.Request) {
    id, err := strconv.Atoi(r.PathValue("id"))
    // ... uses app.snippets, app.logger
}

// Wire it in main()
app := &application{
    logger:   slog.New(slog.NewTextHandler(os.Stdout, nil)),
    snippets: &models.SnippetModel{DB: db},
}

mux.HandleFunc("GET /snippets/{id}", app.snippetView)

This pattern makes testing easy: create an application with fake dependencies (no real DB, no real logger) and call handler methods directly.

Middleware

Middleware in Go is a function that wraps a Handler and returns a new Handler. This simple pattern composes cleanly into chains without a framework.

The Middleware Pattern

// Middleware signature: takes a Handler, returns a Handler
func myMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // Code here runs BEFORE the handler
        next.ServeHTTP(w, r)
        // Code here runs AFTER the handler
    })
}

// Apply: wrap the mux with middleware
http.ListenAndServe(":4000", myMiddleware(mux))

Logging Middleware

func (app *application) logRequest(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        app.logger.Info("request",
            "method", r.Method,
            "uri",    r.URL.RequestURI(),
            "remote", r.RemoteAddr,
            "dur",    time.Since(start),
        )
    })
}
This naive version can't log the status code because ResponseWriter doesn't expose it after the fact. In production, wrap ResponseWriter with a custom type that captures the code (see below).

Panic Recovery

An unrecovered panic in a goroutine crashes only that goroutine. In a Go HTTP server, each request runs in its own goroutine — so an unrecovered panic in a handler closes the connection without sending an HTTP error, and logs "recovered from panic" at most. Catch it explicitly:

func (app *application) recoverPanic(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if err := recover(); err != nil {
                // Connection: close tells the client this connection won't be reused
                w.Header().Set("Connection", "close")
                app.serverError(w, r, fmt.Errorf("%s", err))
            }
        }()
        next.ServeHTTP(w, r)
    })
}

Security Headers

func secureHeaders(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Content-Security-Policy",
            "default-src 'self'; style-src 'self' fonts.googleapis.com")
        w.Header().Set("Referrer-Policy", "origin-when-cross-origin")
        w.Header().Set("X-Content-Type-Options", "nosniff")
        w.Header().Set("X-Frame-Options", "deny")
        w.Header().Set("X-XSS-Protection", "0") // modern browsers use CSP instead
        next.ServeHTTP(w, r)
    })
}

Capturing Status Codes with a ResponseWriter Wrapper

type responseWriter struct {
    http.ResponseWriter
    status      int
    wroteHeader bool
}

func (rw *responseWriter) WriteHeader(code int) {
    if !rw.wroteHeader {
        rw.status = code
        rw.wroteHeader = true
        rw.ResponseWriter.WriteHeader(code)
    }
}

func (app *application) logRequest(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        wrapped := &responseWriter{ResponseWriter: w, status: http.StatusOK}
        next.ServeHTTP(wrapped, r)
        app.logger.Info("request",
            "method", r.Method,
            "uri",    r.URL.RequestURI(),
            "status", wrapped.status,
            "dur",    time.Since(start),
        )
    })
}

Composing Middleware Chains

// routes.go — wire everything together
func (app *application) routes() http.Handler {
    mux := http.NewServeMux()

    // Static files — no auth, no session
    mux.Handle("GET /static/", http.FileServer(http.Dir("./ui/static")))

    // All other routes get the full middleware stack
    mux.HandleFunc("GET /", app.home)
    mux.HandleFunc("GET /snippets/{id}", app.snippetView)
    mux.HandleFunc("POST /snippets", app.requireAuth(app.snippetCreate))

    // Apply global middleware: outermost = first to run
    return app.recoverPanic(
        app.logRequest(
            secureHeaders(mux),
        ),
    )
}
For longer chains, many teams extract a helper like alice or write their own small Chain(middlewares ...func(http.Handler) http.Handler) function. The pattern is always the same.

Template Caching

Parsing templates on every request is expensive. Parse once at startup, cache in memory:

// Build a map: page name → parsed template set
func newTemplateCache() (map[string]*template.Template, error) {
    cache := map[string]*template.Template{}

    // Glob all page templates
    pages, err := filepath.Glob("./ui/html/pages/*.html")
    if err != nil {
        return nil, err
    }

    for _, page := range pages {
        name := filepath.Base(page)

        // Parse base first, then partials, then the page itself
        ts, err := template.New(name).Funcs(functions).ParseFiles(
            "./ui/html/base.html",
            "./ui/html/partials/*.html",
            page,
        )
        if err != nil {
            return nil, err
        }

        cache[name] = ts
    }
    return cache, nil
}

// In main(), store on the application struct
app.templateCache, err = newTemplateCache()

Rendering Templates Safely

Write to a buffer first — if template execution fails mid-render, you've already written the 200 header and you can't take it back. Buffer it so you can return a proper 500 instead:

func (app *application) render(w http.ResponseWriter, r *http.Request,
    status int, page string, data templateData) {

    ts, ok := app.templateCache[page]
    if !ok {
        app.serverError(w, r, fmt.Errorf("template %q not found", page))
        return
    }

    // Render into a buffer first — catches runtime errors before we write status
    buf := new(bytes.Buffer)
    err := ts.ExecuteTemplate(buf, "base", data)
    if err != nil {
        app.serverError(w, r, err)
        return
    }

    w.WriteHeader(status)
    buf.WriteTo(w) // only write to w if no error
}

Template Data

Pass a single struct to templates so you can add fields without changing all call sites:

type templateData struct {
    CurrentYear int
    Snippet     *models.Snippet
    Snippets    []*models.Snippet
    Form        snippetCreateForm // for re-displaying form with errors
    Flash       string            // one-time success message from session
    IsAuthenticated bool
}

// Populate common fields automatically
func (app *application) newTemplateData(r *http.Request) templateData {
    return templateData{
        CurrentYear:     time.Now().Year(),
        Flash:           app.sessionManager.PopString(r.Context(), "flash"),
        IsAuthenticated: app.isAuthenticated(r),
    }
}

database/sql

Go's database/sql package provides a unified interface to any SQL database. You bring a driver (e.g., go-sql-driver/mysql, lib/pq, jackc/pgx) and the stdlib handles connection pooling, prepared statements, and transactions.

Opening and Configuring the Pool

import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql" // side-effect import: registers the driver
)

func openDB(dsn string) (*sql.DB, error) {
    db, err := sql.Open("mysql", dsn)
    if err != nil {
        return nil, err
    }

    // sql.Open doesn't actually connect — ping to verify connectivity
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
    if err = db.PingContext(ctx); err != nil {
        db.Close()
        return nil, err
    }

    // Connection pool tuning
    db.SetMaxOpenConns(25)        // max connections to the DB
    db.SetMaxIdleConns(25)        // max idle connections kept open
    db.SetConnMaxLifetime(5 * time.Minute) // recycle connections

    return db, nil
}
sql.DB is a connection pool, not a single connection. It's safe to use concurrently. Create one at startup, share it — don't create a new one per request.

Query Patterns

Three methods cover most query use cases:

// QueryRowContext — exactly one row expected
func (m *SnippetModel) Get(id int) (*Snippet, error) {
    stmt := `SELECT id, title, content, created, expires
             FROM snippets WHERE id = ? AND expires > UTC_TIMESTAMP()`

    s := &Snippet{}
    err := m.DB.QueryRowContext(context.Background(), stmt, id).Scan(
        &s.ID, &s.Title, &s.Content, &s.Created, &s.Expires,
    )
    if err != nil {
        if errors.Is(err, sql.ErrNoRows) {
            return nil, ErrNoRecord // sentinel error, not sql.ErrNoRows
        }
        return nil, err
    }
    return s, nil
}

// QueryContext — multiple rows
func (m *SnippetModel) Latest() ([]*Snippet, error) {
    stmt := `SELECT id, title, content, created, expires
             FROM snippets ORDER BY created DESC LIMIT 10`

    rows, err := m.DB.QueryContext(context.Background(), stmt)
    if err != nil {
        return nil, err
    }
    defer rows.Close() // ALWAYS defer Close — releases connection back to pool

    var snippets []*Snippet
    for rows.Next() {
        s := &Snippet{}
        err := rows.Scan(&s.ID, &s.Title, &s.Content, &s.Created, &s.Expires)
        if err != nil {
            return nil, err
        }
        snippets = append(snippets, s)
    }
    // rows.Err() catches errors that happened during iteration
    if err = rows.Err(); err != nil {
        return nil, err
    }
    return snippets, nil
}

// ExecContext — INSERT, UPDATE, DELETE (no rows returned)
func (m *SnippetModel) Insert(title, content string, expires int) (int, error) {
    stmt := `INSERT INTO snippets (title, content, created, expires)
             VALUES(?, ?, UTC_TIMESTAMP(), DATE_ADD(UTC_TIMESTAMP(), INTERVAL ? DAY))`

    result, err := m.DB.ExecContext(context.Background(), stmt, title, content, expires)
    if err != nil {
        return 0, err
    }
    id, err := result.LastInsertId()
    return int(id), err
}

Transactions

func (m *Model) transfer(fromID, toID, amount int) error {
    tx, err := m.DB.BeginTx(ctx, nil)
    if err != nil {
        return err
    }
    // If anything goes wrong, rollback. Rollback after Commit is a no-op.
    defer tx.Rollback()

    _, err = tx.ExecContext(ctx, "UPDATE accounts SET balance = balance - ? WHERE id = ?", amount, fromID)
    if err != nil {
        return err
    }
    _, err = tx.ExecContext(ctx, "UPDATE accounts SET balance = balance + ? WHERE id = ?", amount, toID)
    if err != nil {
        return err
    }

    return tx.Commit() // only commits if we reach this line
}

Forms, Validation & Sessions

Processing user input — parsing forms, validating data, and persisting state between requests — is the bread-and-butter of web applications. Go handles all of it without a framework.

Form Parsing

func (app *application) snippetCreate(w http.ResponseWriter, r *http.Request) {
    // ParseForm populates r.Form and r.PostForm
    err := r.ParseForm()
    if err != nil {
        app.clientError(w, http.StatusBadRequest)
        return
    }

    // r.PostForm.Get returns "" for missing keys (safe, no panic)
    title   := r.PostForm.Get("title")
    content := r.PostForm.Get("content")
    expires, err := strconv.Atoi(r.PostForm.Get("expires"))
    if err != nil {
        app.clientError(w, http.StatusBadRequest)
        return
    }
    // ...
}

Validation Pattern

Embed a Validator in your form struct to collect field errors without a library:

type Validator struct {
    FieldErrors    map[string]string // field-level errors
    NonFieldErrors []string          // form-level errors
}

func (v *Validator) Valid() bool {
    return len(v.FieldErrors) == 0 && len(v.NonFieldErrors) == 0
}

func (v *Validator) AddFieldError(key, message string) {
    if v.FieldErrors == nil {
        v.FieldErrors = make(map[string]string)
    }
    if _, exists := v.FieldErrors[key]; !exists {
        v.FieldErrors[key] = message
    }
}

func (v *Validator) CheckField(ok bool, key, message string) {
    if !ok {
        v.AddFieldError(key, message)
    }
}

// Form struct embeds Validator
type snippetCreateForm struct {
    Title   string
    Content string
    Expires int
    Validator
}

// In the handler — validate and re-render on failure
form := snippetCreateForm{
    Title:   r.PostForm.Get("title"),
    Content: r.PostForm.Get("content"),
    Expires: expires,
}
form.CheckField(form.Title != "", "title", "This field cannot be blank")
form.CheckField(len(form.Title) <= 100, "title", "Must be 100 characters or fewer")

if !form.Valid() {
    data := app.newTemplateData(r)
    data.Form = form
    app.render(w, r, http.StatusUnprocessableEntity, "create.html", data)
    return
}

In the template, display per-field errors inline: {{if .Form.FieldErrors.title}}{{.Form.FieldErrors.title}}{{end}}

Session Management

Sessions store user state server-side across requests. The popular alexedwards/scs package wraps cookie-based sessions with multiple backend options:

import "github.com/alexedwards/scs/v2"

// In main() — configure once
sessionManager := scs.New()
sessionManager.Store = mysqlstore.New(db)  // or: memstore, redisstore
sessionManager.Lifetime = 12 * time.Hour
sessionManager.Cookie.Secure = true       // HTTPS only

app := &application{
    sessionManager: sessionManager,
}

// All routes must be wrapped with the session middleware
return sessionManager.LoadAndSave(app.routes())

// In handlers — get, set, pop values by string key
// Pop atomically reads and deletes — for one-time flash messages
flash := app.sessionManager.PopString(r.Context(), "flash")

app.sessionManager.Put(r.Context(), "flash", "Snippet created!")
app.sessionManager.Put(r.Context(), "authenticatedUserID", id)

userID := app.sessionManager.GetInt(r.Context(), "authenticatedUserID")
Always regenerate the session token after login (sessionManager.RenewToken) to prevent session fixation attacks.

CSRF Protection

Any state-changing form needs a CSRF token. Use gorilla/csrf or the simpler justinas/nosurf:

import "github.com/justinas/nosurf"

// Wrap the router with the CSRF middleware
func noSurf(next http.Handler) http.Handler {
    csrfHandler := nosurf.New(next)
    csrfHandler.SetBaseCookie(http.Cookie{
        HttpOnly: true,
        Path:     "/",
        Secure:   true, // HTTPS only
    })
    return csrfHandler
}

// Pass the token to every template
func (app *application) newTemplateData(r *http.Request) templateData {
    return templateData{
        CSRFToken: nosurf.Token(r),
        // ...
    }
}

// In the form template
<input type="hidden" name="csrf_token" value="{{.CSRFToken}}">

TLS, HTTPS & File Embedding

Go makes HTTPS trivially easy. The standard library includes a full TLS stack. In production you'll usually terminate TLS at a reverse proxy (Caddy, Nginx), but knowing how to do it in Go matters for internal services.

HTTPS with ListenAndServeTLS

srv := &http.Server{
    Addr:    ":4000",
    Handler: app.routes(),
    // TLS config — restrict to modern cipher suites
    TLSConfig: &tls.Config{
        CurvePreferences: []tls.CurveID{tls.X25519, tls.CurveP256},
        MinVersion:       tls.VersionTLS12,
    },
    // With TLS, timeouts are especially important — slow TLS handshakes can pile up
    ReadTimeout:  5 * time.Second,
    WriteTimeout: 10 * time.Second,
    IdleTimeout:  60 * time.Second,
}

// cert.pem and key.pem — in dev, generated with:
// go run $(go env GOROOT)/src/crypto/tls/generate_cert.go --rsa-bits=2048 --host=localhost
log.Fatal(srv.ListenAndServeTLS("./tls/cert.pem", "./tls/key.pem"))

go:embed — Embedding Files at Compile Time

The //go:embed directive (Go 1.16+) bundles files into the binary at compile time. The result is a single self-contained binary with no external file dependencies — essential for containers and easy deployment:

import "embed"

// Embed a single file
//go:embed ui/html/base.html
var baseTemplate string

// Embed a directory as an fs.FS
//go:embed ui/html
var htmlFiles embed.FS

//go:embed ui/static
var staticFiles embed.FS

// Serve static files from embedded FS
staticFS, _ := fs.Sub(staticFiles, "ui/static")
mux.Handle("GET /static/", http.FileServer(http.FS(staticFS)))

// Parse templates from embedded FS
func newTemplateCache() (map[string]*template.Template, error) {
    cache := map[string]*template.Template{}

    pages, err := fs.Glob(htmlFiles, "ui/html/pages/*.html")
    if err != nil {
        return nil, err
    }

    for _, page := range pages {
        name := filepath.Base(page)
        // Must list patterns in the order: base, partials, page
        patterns := []string{
            "ui/html/base.html",
            "ui/html/partials/*.html",
            page,
        }
        ts, err := template.New(name).Funcs(functions).ParseFS(htmlFiles, patterns...)
        if err != nil {
            return nil, err
        }
        cache[name] = ts
    }
    return cache, nil
}
The //go:embed directive must appear immediately before the variable declaration with no blank line. The path is relative to the source file's directory. Directories are embedded recursively; files beginning with . or _ are excluded unless you use all: prefix.

Authentication Flow

Cookie-based auth pattern — stores only the user ID in the session, not credentials:

// Signup — hash the password, never store plaintext
func (app *application) userSignup(w http.ResponseWriter, r *http.Request) {
    // ... parse and validate form ...

    hashedPassword, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
    if err != nil {
        app.serverError(w, r, err)
        return
    }
    err = app.users.Insert(name, email, string(hashedPassword))
    // handle duplicate email (check for specific DB error)...
}

// Login — verify and put userID in session
func (app *application) userLogin(w http.ResponseWriter, r *http.Request) {
    id, err := app.users.Authenticate(email, password)
    if err != nil {
        if errors.Is(err, models.ErrInvalidCredentials) {
            form.AddNonFieldError("Email or password is incorrect")
            // re-render form
            return
        }
        app.serverError(w, r, err)
        return
    }

    // Renew session token on privilege level change (session fixation defense)
    err = app.sessionManager.RenewToken(r.Context())
    if err != nil {
        app.serverError(w, r, err)
        return
    }

    app.sessionManager.Put(r.Context(), "authenticatedUserID", id)
    http.Redirect(w, r, "/snippets/create", http.StatusSeeOther)
}

// Model — authenticate returns the user ID
func (m *UserModel) Authenticate(email, password string) (int, error) {
    var id int
    var hashedPassword []byte
    err := m.DB.QueryRowContext(ctx,
        "SELECT id, hashed_password FROM users WHERE email = ?", email,
    ).Scan(&id, &hashedPassword)
    if errors.Is(err, sql.ErrNoRows) {
        return 0, ErrInvalidCredentials
    }

    err = bcrypt.CompareHashAndPassword(hashedPassword, []byte(password))
    if errors.Is(err, bcrypt.ErrMismatchedHashAndPassword) {
        return 0, ErrInvalidCredentials
    }

    return id, nil
}

// requireAuth middleware — redirect unauthenticated users
func (app *application) requireAuth(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        if !app.isAuthenticated(r) {
            http.Redirect(w, r, "/user/login", http.StatusSeeOther)
            return
        }
        // Prevent authenticated pages from being stored in browser cache
        w.Header().Add("Cache-Control", "no-store")
        next.ServeHTTP(w, r)
    })
}
